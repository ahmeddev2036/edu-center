# SRS (اختصار)
- النطاق: إدارة طلاب، حضور، مالية، موظفين، امتحانات، فيديوهات، صلاحيات.
- الأطراف: مدير السنتر، المعلم، ولي الأمر، الطالب، مسؤول مالي، مسؤول النظام.
- قيود: SLA 99.5%، زمن استجابة واجهات 300ms للعمليات الأساسية، دعم حتى 2036 مع قابلية تبديل المكونات.
- متطلبات عليا: مصادقة 2FA، RBAC مرن، تقارير مالية، تكامل واتساب، تكامل ZKTeco، فيديوهات مؤمنة، تطبيق جوال.

## حالات استخدام مختصرة
- الطالب: عرض جدول/حضور/نتائج/مدفوعات، مشاهدة فيديوهات مصرح بها.
- المعلم: تسجيل حضور، رفع درجات، إنشاء امتحانات، رفع فيديو.
- المالي: تسجيل قبض/صرف، ربط دفعات بالحضور والأقساط، تقارير.
- المدير: إدارة صلاحيات، مراقبة الأداء، إدارة المستخدمين.

## قيود غير وظيفية
- أمن: تشفير بيانات حساسة AES-256، TLS، Rate limiting.
- أداء: دعم آلاف الطلاب المتزامنين، كاش Redis، Queue للرسائل.
- توفّر: نشر على Kubernetes مع مراقبة (Prometheus/Grafana) وتتبع أخطاء (Sentry).
